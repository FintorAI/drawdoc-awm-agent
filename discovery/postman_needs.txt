(4) eDisclosure - Opening Package (Initial Disclosure)

# Step 1: Audit loan before ordering
POST /encompassdocs/v1/documentAudits/opening
Body: { "loanId": "{{loanId}}" }
→ Returns audit issues (if any) - BLOCKING if issues exist

# Step 2: Generate opening (Initial Disclosure) doc set
POST /encompassdocs/v1/documentOrders/opening
Body: { 
  "loanId": "{{loanId}}",
  "consumerConnectSiteId": "{{siteId}}"
}
→ Returns docSetId

# Step 3: Send package to borrower
POST /encompassdocs/v1/documentOrders/opening/{docSetId}/delivery
→ Sends to borrower via Consumer Connect
→ Creates Disclosure Tracking entry

# Optional: Get status
GET /encompassdocs/v1/documentOrders/opening/{docSetId}/status


(5) ATR/QM
Context: V3 Get Document API - Updated
The V3 Get Document API has been updated to retrieve ATR/QM document verification details such as employment, income, obligations and assets for a given document.

In addition, a new retrieve only attribute documentStatus has been added to the Document Contract to indicate the status of the document. This attribute is a replacement for the existing status attribute which will be deprecated in a future release.

This update impacts the following endpoint:

V3 Get a Document
GET /v3/loans/{loanId}/documents/{documentId}
