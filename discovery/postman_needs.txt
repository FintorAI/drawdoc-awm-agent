# Get virtual fields (calculated fields)
GET /encompass/v3/schemas/loan/virtualFields
→ Returns list of calculated/virtual fields

# Get full loan schema
GET /encompass/v3/schemas/loan
→ Returns complete loan object structure

# V1 API - Get specific field schema
GET /encompass/v1/schema/loan/{{fieldId}}
→ Returns detailed info for one field

# V1 API - Get JSON path for field IDs
POST /encompass/v1/schema/loan/pathGenerator
Body: ["4000", "4002", "LE1.X1"]
→ Returns the JSON paths in the loan object for each field ID

  python test_all_fields.py --verify                    # Verify all disclosure fields

================================================================================
(7) DISCLOSURE TRACKING ENDPOINTS
================================================================================

Check if disclosures have been sent and their status.

# Get list of all disclosure tracking logs
GET /encompass/v3/loans/{loanId}/disclosureTracking2015Logs
→ Returns array of disclosure tracking entries
→ Each entry has "contents" array: ["LE"], ["CD"], etc.
→ If empty array returned: NO disclosures sent yet → eligible for Initial LE

# Get specific disclosure tracking log
GET /encompass/v3/loans/{loanId}/disclosureTracking2015Logs/{disclosureTrackingId}
→ Returns full details including recipients, eConsent status, etc.

# Get disclosure tracking snapshot (loan state at time of disclosure)
GET /encompass/v3/loans/{loanId}/disclosureTracking2015Logs/{disclosureTrackingId}/snapshot
→ Returns the loan data snapshot at the time disclosure was sent

# Create manual disclosure tracking entry (for LE)
POST /encompass/v3/loans/{loanId}/disclosureTracking2015Logs?applicationId={applicationId}
Body: { "disclosedMethod": "InPerson", "contents": ["LE", "ServiceProviderListNoFee"] }

# Update disclosure tracking (add Intent to Proceed)
PATCH /encompass/v3/loans/{loanId}/disclosureTracking2015Logs/{disclosureTrackingId}
Body: { "intentToProceed": { "intent": true, "date": "2024-12-02" } }

Usage: Test with --disclosure-tracking flag:
  python test_all_fields.py --disclosure-tracking      # Check if LE/CD already sent

================================================================================
(8) MILESTONES ENDPOINTS
================================================================================

Check loan stage/milestone status.

# Get all milestones for loan
GET /encompass/v3/loans/{loanId}/milestones
→ Returns array of milestones with:
   - name: "Started", "Qualification", "Processing", "Approval", "Funding", etc.
   - doneIndicator: true/false (is milestone completed?)
   - startDate, doneDate, expectedDate
   - loanAssociate: user/role assigned

# Key milestones for disclosure eligibility:
   - "Started" or "Qualification" done = loan active, may need Initial LE
   - "Funding" or later done = loan closed, not eligible for disclosure

Usage: Test with --milestones flag:
  python test_all_fields.py --milestones               # Get all milestones
  python test_all_fields.py --pre-check                # Full eligibility check

================================================================================
(9) FULL PRE-CHECK FOR DISCLOSURE ELIGIBILITY
================================================================================

The --pre-check flag runs a comprehensive eligibility check:

1. MILESTONES: Is loan at appropriate stage?
   - Must have "Started" completed
   - Must NOT have "Funding"/"Completion" completed

2. DISCLOSURE TRACKING: Has Initial LE been sent?
   - If no "LE" in contents → eligible for Initial LE
   - If "LE" exists → would be COC/Revised LE (not MVP)

3. TRID DATES: Is loan within compliance window?
   - Application Date (745) must be set
   - LE Due Date (3152) should not be passed

Usage:
  python test_all_fields.py --pre-check                # Run full pre-check
